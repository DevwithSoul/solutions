# Automated PDF Invoice Data Extractor

## Problem Description
Administrative staff often spend hours manually transcribing data (Invoice Numbers, Dates, Totals) from PDF invoices into Excel spreadsheets. This process is slow, tedious, and prone to human error (typos, missed decimals).

## Solution Overview
This tool automates the extraction process using Python. It reads every PDF file in a specified folder, uses pattern matching (Regex) to identify key fields, and compiles the data into a clean Excel spreadsheet.

**Key Features:**
- **Batch Processing:** Handles hundreds of files in seconds.
- **Text Extraction:** Uses `pdfplumber` for accurate text reading.
- **Excel Export:** Automatically generates a formatted `.xlsx` file.
- **Sample Generator:** Includes a feature to create a dummy invoice for testing.

## Prerequisites
- Python 3.7 or higher installed.
- `pip` (Python package manager).

## Installation

1. **Unzip the solution**.
2. **Install dependencies**:
   Open your terminal/command prompt in the folder and run:
   ```bash
   pip install -r requirements.txt
   ```

## Usage

### 1. Basic Usage
Place your PDF invoices in a folder (e.g., `invoices/`) and run:

```bash
python invoice_extractor.py --input invoices
```
This will generate a file named `invoice_data.xlsx` in the current directory.

### 2. Generate a Test Invoice
If you don't have invoices handy, you can generate a sample one to verify the tool works:

```bash
python invoice_extractor.py --input invoices --generate-sample
```

### 3. Custom Output Filename
```bash
python invoice_extractor.py --input invoices --output my_report.xlsx
```

## Configuration

### Adjusting Patterns
The script uses Regular Expressions (Regex) to find data. If your invoices use a different format (e.g., "Inv #:" instead of "Invoice Number:"), you can modify the `REGEX_PATTERNS` dictionary at the top of `invoice_extractor.py`.

**Current Patterns:**
- Invoice: Looks for `Invoice Number:`, `Invoice #:` followed by alphanumeric characters.
- Date: Looks for `YYYY-MM-DD` or `MM/DD/YYYY`.
- Total: Looks for `Total:` or `Total Amount:` followed by currency.

## Recommendations
- **Scanned PDFs:** This tool works best on "native" PDFs (generated by software). For scanned images (files that are just pictures inside a PDF), OCR (Optical Character Recognition) libraries like `pytesseract` would be required, which is outside the scope of this lightweight version.
- **Validation:** Always spot-check the Excel output against the original PDFs to ensure the patterns match your specific invoice layout.
